System Architecture

The architecture is designed as a clean pipeline with strict layer separation so that data reliability, simulation realism, and reporting speed can scale independently.

Data Sources

GA4 event streams, campaign metadata, and internal business dimensions form the raw input layer used for baseline behavior tracking.

Stack: GA4 API, CSV exports, connector scripts.

Processing

Events are cleaned, standardized, and sessionized to create analytics-ready tables with consistent naming conventions and timestamps.

Stack: Python ETL, SQL transformations.

Simulation

Agent models consume processed behavior patterns to generate synthetic journeys and conversion outcomes across modeled scenarios.

Stack: Python simulation engine, rule-based logic modules.

Analytics

Real and simulated data are combined for funnel metrics, anomaly detection, and comparative scenario analysis.

Stack: SQL marts, metric calculation pipelines.

Dashboard

Decision dashboards surface KPI movements, channel impact, and recommendations for product and growth stakeholders.

Stack: Streamlit, interactive chart components.
